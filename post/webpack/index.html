웹팩 직접 설정하기

시작한 이유

  • 평소에는 react, vue등을 사용할 때에는 굳이 webpack을 설정하고 사용하는적이 매우 드물었다.
  • 그러던 와중에 프레임워크를 좀 더 이해하기 위해 바닐라자바스크립트로 SPA를 만들기를 도전 하던 중 webpack설정에 도전하게 되었다.

웹팩이란

  • 웹팩이란 모듈 번들러이다.
  • 모듈 번들러란 웹 어플리케이션을 구성하는 자원들(HTML, css, js, images ...)을 각각의 모듈로 보고 이를 조합해서 병합된 하나의 결과물을 만드는 도구를 의미한다.

웹팩을 사용하는 이유

웹팩이 등장한 이유는 크게 3가지이다.

  • 파일 단위의 자바스크립트 모듈 관리
  • 웹 개발 작업 자동화 도구
  • 웹 어플리케이션의 빠른 로딩속도와 높은 성능

웹팩 사용해보기

1. 웹팩 설치

shell
1yarn add webpack webpack-cli --dev
2

2. index.html 추가

html
1<!DOCTYPE html>
2<html lang="en">
3  <head>
4    <title>webpack Test</title>
5  </head>
6  <body>
7    <div id="app"></div>
8

9    <script type="module" src="./src/index.js"></script>
10    \
11  </body>
12</html>
13

3. src 추가

javascript
1// hello.js
2const word = 'Helloooooooo';
3

4export default word
5

6// world.js
7const word = 'World';
8

9export default word
10

11// index.js
12import hello_word from "./hello.js"
13import world_word from "./world.js"
14

15document.querySelector('#app').innerHTML = hello_word + ' ' + world_word
16

4. webpack.config.js 웹팩 설정하기

  • entry는 빌드를 시작할 때 최초 진입점이라고 생각하면 좋다.
javascript
1module.exports = {
2  entry: {
3    main: './src/index.js',
4  },
5};
6

위 코드를 추가하면 웹팩이 실행될 때 index.js를 빌드한다.

  • output은 말그대로 웹팩 결과물 파일경로를 의미한다.
javascript
1const path = require('path');
2module.exports = {
3  output: {
4    path: path.resolve('./dist'),
5    filename: '[name].js',
6  },
7};
8
  • loader는 js파일이 아닌 웹 자원들을 변환 할 수 있게 도와주는 속성이다.
javascript
1module.exports = {
2  module: {
3    rules: [
4      {
5        test: /\.js$/,
6        use: 'babel-loader',
7        exclude: /node_modules/,
8      },
9      {
10        test: /\.css$/,
11        use: ['style-loader', 'css-loader'],
12      },
13    ],
14  },
15};
16

이 외에도 sass, file, ts loader 등등 자주 사용되는 로더들이 있다. 사용하는 프로젝트에 필요한 로더가 있을 때 npm에서 다운받아 위 예시처럼 사용하면 된다.

  • 로더는 웹팩 변환할 때 관여를 한다면, plugins는 플러그인 결과물 형태를 바꾸는 역할을 한다.
javascript
1const HtmlWebpackPlugin = require('html-webpack-plugin');
2const { CleanWebpackPlugin } = require('clean-webpack-plugin');
3

4module.exports = {
5  plugins: [
6    new HtmlWebpackPlugin({
7      template: './index.html',
8    }),
9    new CleanWebpackPlugin(),
10  ],
11};
12

위 예시 처럼 사용하면 되는데 HtmlWebpackPlugin => 빌드한 결과물로 HTML 파일을 생성해주는 플러그인 CleanWebpackPlugin => 빌드 이전 결과물을 제거하는 플러그인

  • devServer
    • 웹팩을 dev서버에서 동작을 확인하기 위해 사용한다.
javascript
1const path = require('path');
2

3module.exports = {
4  devServer: {
5    host: 'localhost', // 사용될 호르스트 설정
6    port: 8080, // 접속 포트 설정
7  },
8};
9

이렇게 설정하고

shell
1npx webpack serve
2# http://localhost:8080 에서 확인이 가능하다!
3
